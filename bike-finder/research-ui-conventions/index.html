  2. Slime
    • What UI elements did I see and/or interact with while finding, comparing, selecting, and configuring the specific product?
      • Front Page
        • CTA Button: Leads the slime shop collection.
        • Carousel: Shows all the popular slimes with different attributes and pictures.
        • Search Button: General search that's fixed to the universal nav. Does not have any filtering set. Can search specifcally by name or attribute.
      • Collections/Shop page
        • Can be found through nav bar or CTA.
        • Filter: can filter through the slimes by status and price. There are various sorting methods. Although the site lists stuff such as texture, scent, and level of the slime, this is not used by the filter.
      • Detailed View
        • Simple description that talks about the slime with a pretty image. Also can add multiple to a cart.
    • Have I seen this UI element before in other contexts? Did I know what this element would do before I interacted with it? Why?
      • These are all very common UI elements, so I've seen them before. Furthermore, because they are so common and the fact the elements are used intuitively, the elements behaved exactly as I expected.
      • The only thing that surprised me was the lack of filtering on the search, since that's a convention.
    • What general problem is this element designed to handle?
      • All these elements are designed to help the user find the slime they want and purchase it. I also guess it's used to encourage exploration, especially with the carousel, to entice users to buy more slimes then they might've intended on purchasing.
    • Why was it used in this context? Is it successful?
      • I would say it was successful. Every element is intuitive. However, I feel as the process could definitely improve, such as the expansion of the filtering section.
  3. Toyota Rav4
    • What UI elements did I see and/or interact with while finding, comparing, selecting, and configuring the specific product?
      • Front Page
        • CTA Carousel: Details all the new/trendy car types. Is timed so it swipes through all the carousel items automatically. For each car shown, it gives big numbers so the user can deduce a cool stat.
        • Dropdown Carousel: Details all the cars avaliable based off a specific car type: upcoming vehicles, cars & minivans, trucks, etc. Crossovers & SUVs is where found my product. Each car has two options, explore and build. Below it you can explore all cars.
        • Series of buttons: Not sure the name. All in line called shopping tools. It has build & price, search inventory, local specials, and find a dealer. It opens a lot more options for the user. It reflects some of the global informaton on the navigation.
      • Explore
        • Select: Specifically, selects a color of the RAV4.
        • Extra Details: Images and text advertise the car. Some interesting things include looking at detailed parts of the specs and zooming on the individual parts of the car. You have to scroll to the button of the page to see all the different price options.
      • Build
        • Customize: I have to pick from 6 attributes before I can request a purchase. These are all extra addons including liners on the floor and interior colors. Some of them have extra costs. Interestingly, they are presented in a horizontal nature.
        • Summarize CTA/Get Quote: Pulls a popup that lets me request the car I want. I've never bought a car, so "get quote" was unknown terminology to me. I only clicked on it since it was center and bold.
      • All Vehicles
        • I didn't go onto this page because it was very easy to find what I wanted from the front page.
        • Filter: Very standard filering service of vehicle category, price, miles, seating, and years. No search option. The results bring up cards that are similar to the cards found in the front page.
        • Sort: Collection sort that's standard. Can sort by price, alphabetical order, and MPG.
    • Have I seen this UI element before in other contexts?
      • I've seen the more simple elements in other contexts, but I've rarely seen combined UI elements such as a dropdown carousel. I like this site because of its clean and innovative functionality.
    • Did I know what this element would do before I interacted with it? Why?
      • Although I did encounter elements that I've rarely used before, I feel like I expected most of the behavior just becuase of the clean presentation and also because the elements were built off of familiar UI elements.
    • What general problem is this element designed to handle?
      • The problem of this site is to help its users find a car they want to purchase.
    • Why was it used in this context? Is it successful?
      • My guess is that most of the users on this site are here to purchase a car or learn more about a specific car. Basically, rather than exploring, users know what they want. As such, the front page mostly shows off the vehicles that users will want to buy or learn more about. I'd say it was successful since I quickly found my chosen vehicle on the front page; I didn't have to do any extra searching. Furthermore, there are mny potential user flows that the user can find their car that I didn't explore/detail here just because there is so much functionality on the front page.
  4. Circe by Madeline Miller
    • What UI elements did I see and/or interact with while finding, comparing, selecting, and configuring the specific product?
      • Front Page
        • Dropdown Search: Can search any keyword such as the title, artist, keyword, and ISBN. Can also specify the medium of what you're searching for (books, ebooks, etc).
        • Categories: I will be able to click on these buttons and find works based off categories such as "New Releases". However, if I was looking specifically for a book, this might be confusing if I tried looking for it in a place but I wouldn't know if I'm looking in the right category.
      • Navigation Bar
        • Nested navigation: Has different categories that have their own internal category. Ex. Fiction to Bestsellers to NYT Bestsellers. All this nested information might make navigation clunky.
      • Detailed View
      • Only found through search. Very standard where it lists the price and lets you add to cart/direct purchase.
    • Have I seen this UI element before in other contexts? Did I know what this element would do before I interacted with it? Why?
      • Yes, these are all very standard elements. As such, I had no issue finding my product
    • What general problem is this element designed to handle?
      • Help users find and purchase a book.
    • Why was it used in this context? Is it successful?
      • I think it was successful since I knew what book I wanted to purchase. Basically, I just used the search bar. However, as a book reader, some of the fun is trying to find new books and I feel like the site doesn't really encourage exploration; the exploration is nested within the navigation.

Summary

  • Universal: Nav bars, search bars, filters
  • Different/Unusual: Nothing felt pretty unusual. Honestly, I felt like many of the UI processes could be expanded on.
  • Specific: For my complicated processes, such as buying a car for Toyota, there was a combination UI element.
